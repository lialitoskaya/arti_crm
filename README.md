# Arti CRM

Локальная CRM для обработки чатов, вопросов, отзывов и базы знаний по маркетплейсам.

Текущая сборка: `v103_analytics_ui_polish_2026-06-18`.

## Быстрый запуск

```cmd
cd /d C:\crm_marketplaces
.venv\Scripts\activate
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

После обновления фронтенда делайте `Ctrl+F5`.

## Основные возможности

- Единый список чатов Ozon / Wildberries / Яндекс Маркет.
- Локальное хранение истории в `crm.sqlite3`.
- Ответы менеджеров из CRM.
- Статусы, ответственные, архив.
- База знаний.
- Ozon Reviews и Ozon Questions.
- WB events auto-import с учётом cooldown.
- Ozon deep backfill для старой истории.
- Ozon fast inbox sync для новых и последних чатов.
- Раздел аналитики: обращения по дням, среднее время ответа, закрытые обращения, активность по часам.

## Важная архитектурная идея

CRM должна использовать API маркетплейсов как источник новых событий, но не как постоянное хранилище истории.

Правильная модель:

```text
API маркетплейса → синхронизация → SQLite CRM → интерфейс
```

Старая история, которую CRM уже получила, должна читаться из локальной базы, а не запрашиваться заново у API.

## Важно по безопасности

Не публикуйте CRM напрямую в интернет без дополнительной защиты. API-ключи маркетплейсов должны оставаться только на сервере в `.env`, не во фронтенде.


## Аналитика

В v86 обращение считается как первое входящее сообщение клиента в конкретном чате за локальный день. Повторные сообщения в этот же день не увеличивают количество обращений, но тот же клиент на следующий день считается новым дневным обращением. Подробнее: `docs/ANALYTICS.md`.


## Автоназначение ответственного

Если чат ещё не назначен, первый сотрудник, который успешно отправил ответ клиенту из CRM, автоматически становится ответственным за этот чат. После этого чат появляется во вкладке **«Мои чаты»** у этого сотрудника, а поле **«Ответственный»** в открытом диалоге заполняется автоматически.

Отключение:

```env
CRM_AUTO_ASSIGN_FIRST_RESPONSE=false
```


## Ответственный за чат

- Первый успешный ответ менеджера автоматически назначает его ответственным, если чат ещё не был назначен.
- Ответственного можно вручную переназначить в открытом диалоге.
- Ручное переназначение имеет приоритет: автоназначение не перезаписывает уже выбранного сотрудника.


## Уведомления

Уведомления показываются всплывающими полупрозрачными карточками в правом верхнем углу. Из карточки можно сразу открыть чат или задачу либо скрыть уведомление.


В CRM есть встроенный центр уведомлений в левом меню.

События v90:
- новое входящее сообщение в назначенном чате — ответственному;
- новое входящее сообщение в неназначенном чате — активным менеджерам/админам;
- ручное назначение ответственным — новому ответственному;
- новая задача — назначенному сотруднику.

Чтобы не засыпать операторов уведомлениями при глубокой догрузке истории, старые сообщения не создают уведомления. Порог регулируется:

```env
CRM_NOTIFICATION_RECENT_MESSAGE_HOURS=24
```


## Аналитика исключает служебные сообщения

В расчётах аналитики не учитываются сообщения ботов, поддержки и системные сообщения маркетплейсов. Они остаются в истории чата, но не влияют на:
- количество обращений;
- входящие сообщения;
- пиковые часы;
- сообщения до 10:00 / после 19:00;
- среднее время ответа.


## Проверка аналитики по строкам

В разделе «Аналитика» добавлен блок «Проверка расчёта по часам». Он показывает конкретные чаты и сообщения, которые попали в отчёт как первые обращения клиента за день. Из строки можно открыть чат.

Также доступен API:
`GET /api/analytics/chats/drilldown`
