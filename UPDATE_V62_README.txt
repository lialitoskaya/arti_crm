v62 — исправление парсинга сообщений Wildberries

Проблема:
- WB-чаты могли попадать в CRM, но сообщения не отображались.
- Причина: ответы WB API иногда приходят не в одном формате. Старый парсер ожидал только result/data/chats и мог терять список, если WB возвращал массив или другой контейнер.
- Также события WB могли содержать chatID, message, text, время и replySign во вложенных полях, которые старый парсер не всегда находил.

Исправлено:
- Поддержан ответ WB API в формате массива.
- Улучшен парсинг /api/v1/seller/chats.
- Улучшен парсинг /api/v1/seller/events.
- Добавлен рекурсивный поиск chatID, текста сообщения, времени, автора и replySign.
- lastMessage из списка чатов используется как fallback-сообщение, если events не вернул историю.
- /api/debug/wb?live=1 теперь показывает sample сообщений по первым WB-чатам, чтобы понять, отдаёт ли WB события.
- Версия: v62_wb_messages_parser_fix_2026-06-15.

Важно:
- Если WB возвращает 429, новые сообщения не появятся до окончания cooldown.
- После установки нужно подождать лимит WB или один раз открыть /api/debug/wb?live=1, затем POST /api/sync/wildberries.
