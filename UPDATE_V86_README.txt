v86 — analytics daily-first request counting

Что изменено:
- Обращение теперь считается как первое входящее сообщение клиента в конкретном чате за локальный день.
- Повторные сообщения того же клиента/чата в этот же день не увеличивают количество обращений.
- Если клиент написал снова на следующий день, это считается новым дневным обращением.
- Пиковые часы теперь считаются по первому дневному обращению, а не по каждому входящему сообщению.
- Среднее время ответа, outside-hours, обращения по дням и разбивка по маркетплейсам переведены на эту же систему.
- Сырые входящие сообщения остались отдельным показателем: raw_inbound_messages / inbound_messages.
- docs/ANALYTICS.md обновлён под новую модель.

После установки:
1. Остановить CRM.
2. Распаковать архив в C:\crm_marketplaces с заменой файлов.
3. Запустить CRM.
4. Проверить /api/debug/version — должна быть v86_analytics_daily_first_requests_2026-06-18.
5. Открыть раздел Аналитика и обновить данные.
