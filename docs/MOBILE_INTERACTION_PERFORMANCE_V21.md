# Arti CRM v21 — why mobile buttons felt slow

## Что видно по видео

Интерфейс не просто долго ждёт ответ API. Задержка проявляется на локальных действиях:
- открыть меню чата;
- открыть панель "Задачи по чату" / "Внутренняя заметка" / "Имя клиента";
- нажимать кнопки внутри уже открытого диалога.

Эти действия не должны ждать backend. Значит проблема была не только в `/api/chats` или sync.

## Где была реальная проблема

В предыдущих версиях я в основном смотрела на `openChat()` и подгрузку сообщений. Это была только часть проблемы.

Фактическая причина мобильной медлительности в v20/v19:

1. В мобильном чате каждые несколько секунд работал live-refresh активного диалога.
2. Этот refresh мог совпадать с тапами по кнопкам и меню.
3. Даже если сообщения не менялись, frontend всё равно делал fetch, обновлял header/list state и держал refresh-флаги.
4. На iOS Safari chat UI использовал дорогие CSS-эффекты:
   - `backdrop-filter: blur(...)`;
   - тяжёлые `box-shadow`;
   - fixed/sticky layers поверх длинного scrollable блока;
   - broad transitions.
5. Поэтому меню/панели могли открываться с ощутимой задержкой, хотя сами обработчики логически простые.

## Что исправлено

### 1. Interaction governor

Добавлена функция `pauseMobileChatBackgroundWork()`.

При тапах по кнопкам/селектам/меню внутри мобильного чата она временно блокирует фоновые refresh/sync операции.

### 2. Instant actions

Пункты меню чата на мобильной версии теперь открываются через `pointerdown`, а не ждут обычный `click`.

### 3. Refresh не лезет поверх открытого меню

`refreshCurrentChatMessagesOnly()` теперь пропускает автообновление, если на мобильной версии открыто меню/панель чата или панель шаблонов.

### 4. Меньше фоновой нагрузки

Частота active-chat refresh увеличена с 5 секунд до 9 секунд.

### 5. Убраны дорогие CSS-слои

На мобильной версии внутри чата отключены:
- backdrop blur;
- лишние тяжелые тени;
- transitions для меню/панелей;
- лишние перерисовки через `contain`.

## Что не менялось

- Backend sync v20 сохранён.
- Ozon/Yandex/WB логика не переписывалась в этом фиксе.
- Live-refresh сообщений сохранён, но больше не должен мешать ручным действиям.
