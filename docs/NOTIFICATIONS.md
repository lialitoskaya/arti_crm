# Уведомления

CRM хранит уведомления в таблице `notifications` и показывает их в левом меню.

## События v90

- Новое входящее сообщение в назначенном чате — ответственному.
- Новое входящее сообщение в неназначенном чате — активным менеджерам/админам.
- Ручное назначение ответственным — новому ответственному.
- Новая задача — назначенному сотруднику.

## API

```text
GET  /api/notifications?limit=30
POST /api/notifications/{id}/read
POST /api/notifications/read-all
```

## Защита от дублей

Для уведомлений используется `dedupe_key`. Повторная синхронизация не должна плодить одинаковые уведомления.

## Исторический импорт

Старые сообщения из backfill не создают уведомления. Порог свежести:

```env
CRM_NOTIFICATION_RECENT_MESSAGE_HOURS=24
```


## Яндекс Маркет

Для Яндекса CRM использует статус чата:
- `NEW`, `WAITING_FOR_PARTNER` — чат требует ответа, уведомление допустимо;
- `WAITING_FOR_CUSTOMER`, `FINISHED`, `CLOSED` — уведомление по сообщению не нужно.
