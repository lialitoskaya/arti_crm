# Правила поддержки и доработок

## Перед изменениями

1. Сделать копию проекта/БД.
2. Проверить текущую версию через `/api/debug/version`.
3. Не менять `.env` пользователя без необходимости.
4. Не удалять историю чатов автоматически.

## Проверка перед выдачей архива

```cmd
python tools/check_project.py
```

Минимум:
- все Python-файлы компилируются;
- нет синтаксических ошибок;
- версия сборки обновлена;
- README/документация отражают изменения.

## Правила синхронизации

- New/recent sync должен быть быстрым.
- Deep backfill должен быть отдельным и управляемым.
- Никогда не смешивать тяжёлый архивный импорт с регулярным polling новых чатов.
- Если API отдаёт сомнительный чат — сохранить, но не удалить.

## Правила работы с Ozon

- Не удалять `service/system-looking` чаты по умолчанию.
- Любая фильтрация должна быть обратимой: hide/classify, но не delete.
- Deep backfill запускается вручную или отдельным планировщиком.
- Fast sync должен работать часто и легко.

## Правила работы с WB

- Уважать cooldown.
- Не открывать live debug часто.
- Events импортировать через cursor и auto planner.

## Аналитика

- Расчёты аналитики держать в `app/services/analytics.py`, а не раздувать `main.py`.
- В UI показывать определения метрик, чтобы менеджеры понимали, что именно считается.
- Для точной аналитики закрытий в будущем добавить историю статусов.


## Автоназначение ответственного

Правило v88:

```text
первый успешный исходящий ответ из CRM в неназначенном чате → назначить текущего пользователя ответственным
```

Ограничения:
- существующий `assigned_user_id` не перезаписывается;
- назначение выполняется только после успешной отправки сообщения в маркетплейс;
- если отправка вернула ошибку, ответственный не меняется;
- отключается через `CRM_AUTO_ASSIGN_FIRST_RESPONSE=false`.

Связанная функция: `repo.assign_chat_to_user_if_unassigned`.


## Ответственный за чат

- Если чат не назначен, первый успешный ответ менеджера автоматически назначает этого менеджера ответственным.
- Ручное переназначение через поле `Ответственный` всегда имеет приоритет.
- Автоназначение не перезаписывает уже назначенного ответственного.
- Если ответственного вручную очистили, следующий успешный ответ снова может назначить сотрудника автоматически.
